森野小屋 V6.4｜存储安全修复

本版针对手机浏览器“空间不足、保存失败、重复生成日记”的问题进行修复：

1. 日记与照片从容量很小的 localStorage 迁移到 IndexedDB 大容量本地存储。
2. 第一次打开 V6.4 时会自动迁移旧日记和照片；localStorage 仅保留轻量索引。
3. 保存采用“先写成功、再更新页面”的事务式流程。写入失败时不会先把重复日记塞进时光轴。
4. 保存按钮处理期间会锁定，防止连续点击产生多份副本。
5. 本机存储异常且已登录云端时，尝试把当前内容直接交给云端兜底。
6. 云端恢复改为等待大容量本地写入完成，不再因 localStorage 图片容量不足而恢复失败。
7. 网络错误 TypeError: Load failed 会明确提示切换网络重试。
8. 新上传照片采用自适应压缩，降低云端数据包和备份体积。
9. 自动请求浏览器持久存储权限，降低系统清理网页数据的概率（是否授予由浏览器决定）。

上线前测试重点：
- 旧 V6.3 数据自动迁移后，日记/照片数量不变。
- 新增一篇带多图日记，只生成一篇。
- 连点保存按钮，不出现重复项。
- 手机刷新后图片仍存在。
- 手机与电脑云端数字一致。
- 从云端恢复到新设备后，图片位置和顺序正常。

注意：V6.4 解决本机浏览器容量问题，但大量照片长期增长后，Supabase 单行 JSON 和历史备份仍可能变大。后续建议升级为 Supabase Storage 独立图片仓库。
